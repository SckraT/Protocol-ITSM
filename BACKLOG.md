# Backlog — улучшения UX/UI

Список предложений по развитию интерфейса. Отметки: `[ ]` — в планах, `[x]` — сделано
(см. [CHANGELOG.md](CHANGELOG.md)). Сгруппировано по трудозатратам.

## 🟢 Быстрые победы

- [x] **1. Тосты вместо `alert()`** — всплывающие уведомления успех/ошибка. *(0.5.0)*
- [x] **2. Кастомный диалог подтверждения** вместо `confirm()`. *(0.5.0)*
- [x] **3. Обработка ошибок и состояний загрузки** — спиннер, «Повторить», тосты при сбое. *(0.5.0)*
- [x] **4. Относительные сроки** — «просрочено на N дней», «сегодня», «через N дней». *(0.5.0)*
- [x] **5. Доступность** — буква в маркере приоритета (В/С/Н), `role="button"`, фокус, `Esc`. *(0.5.0)*

## 🟡 Среднее

- [x] **6. Сводка-дашборд** над таблицей: «В работе · Просрочено · Закрыто за неделю». *(0.6.0)*

- [x] **7. Расширенные фильтры.** Сейчас фильтр только по состоянию. Добавить быстрый
  отбор по **отделу / исполнителю / приоритету** и «только просроченные».
  Частый запрос при разборе протокола. *(0.7.0)*

- [x] **8. Запоминание вида.** Фильтр, поиск и сортировка сбрасываются при перезагрузке.
  Сохранять в URL или `localStorage`. Хорошо дополняет #7. *(0.7.1)*

- [ ] **9. Оптимистичный UI вместо полного перезапроса.** Каждое действие зовёт `loadAll()`
  и перерисовывает всю таблицу — теряется позиция прокрутки и раскрытые строки аккордеона.
  Точечное обновление DOM = ощутимо «живее».

- [x] **10. Печать / отчёт.** Протокол часто нужно показать на совещании или сохранить в PDF.
  Сейчас только CSV. Print-стили (`@media print`) дали бы аккуратную печатную версию
  почти бесплатно. *(0.9.1)*
